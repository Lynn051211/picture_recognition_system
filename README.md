# 图像识别系统

基于 PyTorch 和 tkinter 的本地图像识别应用，点击图片框选择、拖拽上传、自动识别并展示 Top-5 分类结果。

## 运行环境

| 项目 | 要求 | 说明 |
|------|------|------|
| 操作系统 | Windows 10/11 | 拖拽功能依赖 windnd（Windows 消息机制） |
| Python | **3.12** | 必须 3.12，3.13/3.14 无 PyTorch 预编译 wheel |
| 硬件 | 普通 PC（无需 GPU） | 纯 CPU 推理，模型约 14MB，内存占用 < 500MB |

> **为什么只能是 Python 3.12？**  
> PyTorch 官方仅对 3.9–3.12 提供 Windows CPU wheel。3.13 和 3.14 不在支持列表中，pip 安装会直接报错。  
> 如果本地已有 3.12 但未设为默认，推荐通过 `winget` 安装并创建项目级 `.venv`：
> ```bash
> winget install Python.Python.3.12 -l "D:\environment\python\Python312"
> python3.12 -m venv .venv
> .venv\Scripts\activate
> ```

## 技术栈

| 组件 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 深度学习框架 | PyTorch + torchvision | 2.12 + 0.27 | CPU-only 推理，无需 GPU |

| 多线程 | threading (标准库) | — | 模型加载和推理在后台线程，不阻塞 UI |
| 标签翻译 | vba34520/ImageNet_Chinese_Translation | — | GitHub 开源中文映射 + 内置 200+ 高频词兜底 |

## 快速开始

```bash
# 1. 克隆仓库
git clone https://github.com/<your-username>/picture_recognition_system.git
cd picture_recognition_system

# 2. 创建虚拟环境（Python 3.12）
python3.12 -m venv .venv
.venv\Scripts\activate

# 3. 安装依赖（CPU 版本）
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu
pip install Pillow windnd

# 4. 启动
python main.py
```

**首次运行流程：**
1. 自动下载 ImageNet 1000 类英文标签 → `model_cache/imagenet_classes.txt`
2. 从 GitHub 拉取中文翻译映射 → `model_cache/imagenet_classes_zh.json`（网络不可用时用内置 200+ 高频词兜底）
3. PyTorch 下载 MobileNetV2 预训练权重（约 14MB）→ `~/.cache/torch/`

## 功能说明

- **图片加载**：点击图片预览框选择图片，或直接拖拽图片文件到预览区域
- **自动识别**：拖入图片后自动触发识别；点击选择的图片需手动点击「开始识别」
- **结果展示**：右侧面板显示 Top-5 识别结果（横条卡片），包含排名徽章、中文类别名、英文原名和置信度进度条
- **支持格式**：JPG、JPEG、PNG、BMP、GIF、WebP、TIFF

## 关键技术细节

### 图像预处理（recognizer.py）

使用 MobileNetV2 官方训练预处理管线，与论文完全一致：

```python
MobileNet_V2_Weights.IMAGENET1K_V1.transforms()
# 内部等价于：
# Resize(232) → CenterCrop(224) → ToTensor → Normalize(mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225])
```

而非手动 `Resize(256) → CenterCrop(224)`——后者会导致识别准确率下降 2–5 个百分点。

**EXIF 方向自动校正**：手机/相机拍摄的照片通常带有 EXIF Orientation 标签，PIL 默认不处理。代码通过 `ImageOps.exif_transpose()` 自动旋转到正确方向，否则竖拍照片识别率极低。

### 中文标签策略

`recognizer.py` 内置了 200+ 个高频 ImageNet 类别的中文翻译作为兜底映射（狗、猫、鸟、车、乐器等常见类）。首次运行时会尝试从 GitHub 下载完整的 1000 类中文翻译，网络不可用时自动降级到内置映射。

### 拖拽实现

使用 `windnd` 替代常见的 `tkinterdnd2`。原因：`tkinterdnd2` 依赖 OLE 注册，在多版本 Python 环境下容易因 COM 冲突显示红色禁止图标或直接失败。`windnd` 通过 ctypes 调用 Windows Shell API（`DragAcceptFiles` + `WM_DROPFILES`），无需注册，兼容性更好。

### 多线程设计

```
主线程 (tkinter mainloop)
  ├── 后台线程 1：模型加载（load_model + load_labels）
  │   └── 完成后更新状态栏 "就绪"
  └── 后台线程 2：推理（predict）
      └── 完成后 root.after(0, callback) 切回主线程更新 UI
```

所有耗时操作（模型加载、推理）均不在主线程执行，保证 GUI 不卡顿。UI 更新通过 `root.after()` 安全地切回主线程。

## 常见问题

**Q: 为什么我的红苹果被识别成"青苹果"甚至"iPod"？**  
A: ImageNet-1K 训练集中根本没有红苹果。这是模型能力的上限，不是代码 bug。1000 个类别中水果仅有约 10 种，大量常见水果不在范围内。

**Q: 拖拽没反应？**  
A: 确保以普通用户权限运行（非管理员），windnd 在管理员模式下可能无法接收拖放消息。

**Q: 能换成其他模型吗？**  
A: 可以。修改 `recognizer.py` 中的模型加载部分，替换为 `resnet50`、`efficientnet` 等任意 torchvision 预训练模型。注意同步修改预处理管线（使用对应 weights 的 `.transforms()`）。

**Q: 能支持 GPU 加速吗？**  
A: 可以。将 `recognizer.py` 中的 `torch.device("cpu")` 改为 `torch.device("cuda")`，然后 `pip install torch` 时去掉 `--index-url` 参数安装 CUDA 版本即可。
